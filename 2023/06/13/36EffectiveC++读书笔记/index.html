Effective C++ 读书笔记

Updated2023-06-17|C++书常看常新
|Word count:1.1k|Reading time:4min|Post View:

一、让自己习惯 C++

上来就是一个条款，让我把 C++ 当成一个语言联邦。

1.1 视 C++ 为一个语言联邦

1.2 尽量以 const, inline, enum 替换 #define

#define 使用起来不容易 debug，也没有封装性，没有办法被封装在 class 中。

而且对于像我这样喜欢把比较运算写成宏的人，也更推荐用 inline 加模板的方式解决。

其对于任意 class 也是可以使用的，只要重载 +-*/ 就行了。

1.4 确保对象被使用前已初始化

在构造函数中保证用初始化列表把每个成员变量都初始化完成（其实只要保证在每个自定义类中完成各自变量的初始化就行了）。

关于 static，由于 non-local static 对象的初始化顺序是不定的，所以可以通过调用 local-static 来保证对象初始化的顺序。

所谓的 Singleton-Pattern 就是如此（应该是懒汉加载）。

以此保证在调用 directory 时，一定存在 fileSystem。

二、构造/析构/赋值运算

2.1 对于第一无二的类，使用 Uncopyable 阻止编译器生成拷贝函数

以上方法可以保证在编译阶段就发现拷贝错误，而不用等到 link 阶段。

2.2 为多态基类声明 virtual 析构函数

如果想要抽象 class 却没有纯虚函数，就将析构函数声明为纯虚函数，反之如果没有虚函数查询 RTI（Runtime Type Information），就将析构函数声明为虚函数。

对于非基类，不要为其声明虚函数，因为存储虚类时会多存储一个 vptr 指向该类专属的 vtbl（增加内存开销）。

2.3 别让异常逃离析构函数

以上可以保证即使用户操作错误也不会抛出异常（可以提醒用户手动解决），不抛出异常便不会影响程序的结束。

2.4 不要让 copy assignment 和 assigment 互相调用

copy assignment 是在（拷贝）构造一个不存在的对象，assigment 是在给一个存在的对象赋值。

为了减少代码膨胀，可以共同调用 init() 实现初始化。

三、资源管理

资源管理类就像驱动器一样，也许只有一个（unique_ptr），也许有多个，还有可能要上锁，需要好好设计。

四、设计与声明

4.1 设计 class

  1. 拆分功能（需要什么功能就引用对应的头文件）；
  2. 运用泛型；
  3. ……

4.3 考虑写出一个不抛异常的 swap 函数

理一下 C++ 编译器的查找逻辑：

  1. 发现 Call File 中的 swap() 调用；
  2. 全局查找 swap()（没找到对应的）；
  3. 查找 T 所在的 namespace 中的 swap()（找到）；
  4. namespace WidgetStuff 中的 swap 函数调用 Widget 的 swap()；
  5. 使用 std::swap()。

以上便能实现对 swap() 的偏特化。

五、实现

5.1 延后变量定义式出现的时间

看两组代码。

分析一下两组代码的成本：

第一组：1 个构造函数 + 1 个析构函数 + n 个赋值操作。

第二组：n 个构造函数 + n 个析构函数。

除非赋值操作的成本显著低于构造 + 析构的成本，否则应该采用第二组的写法，以增强可读性。
